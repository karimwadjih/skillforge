# Changelog Composer

Compose a clear changelog entry set from commits, PRs, or release notes.

## Quick View

- Category: `docs`
- Maturity: `experimental`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Changelogs often overfit engineering language and miss what users need to know.

## When To Reach For It

- Release changes need a human-readable changelog.
- Teams need a changelog draft from commits or PR summaries.

## What You Should Provide

- change list
- audience
- version context

## What You Should Get Back

- changelog draft
- highlights
- upgrade notes

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is part of the seeded breadth layer. It is intentionally lighter than the developed and flagship tiers.
