# Edge Cases

## Inputs conflict

### Risk

Different stakeholders provide conflicting inputs for support playbook writer.

### How The Skill Should Respond

Surface the disagreement explicitly, state which interpretation drives the output, and list follow-up questions.

### Failure Probe

The response ignores the complication or hides the uncertainty instead of adapting the workflow.

## Evidence is thin

### Risk

The workflow is still worth running, but the evidence is incomplete or uneven.

### How The Skill Should Respond

Reduce certainty, mark assumptions, and preserve the structure of the final artifact without pretending the evidence is stronger than it is.

### Failure Probe

The response ignores the complication or hides the uncertainty instead of adapting the workflow.

## Stakeholder ambiguity

### Risk

The audience for the output is not fully clear.

### How The Skill Should Respond

Choose a default audience, label it, and include a note on how the artifact should change for a different audience.

### Failure Probe

The response ignores the complication or hides the uncertainty instead of adapting the workflow.
