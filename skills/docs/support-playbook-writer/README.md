# Support Playbook Writer

Write a support playbook for recurring issue types and escalation routes.

## Quick View

- Category: `docs`
- Maturity: `experimental`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Support organizations often repeat the same work because playbooks are thin or outdated.

## When To Reach For It

- Support handling is inconsistent for a recurring issue type.
- A support playbook needs to be drafted from scattered procedures.

## What You Should Provide

- issue types
- resolution notes
- escalation rules

## What You Should Get Back

- support playbook
- handling guidance
- escalation map

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is part of the seeded breadth layer. It is intentionally lighter than the developed and flagship tiers.
