# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Clarity | The output is direct, structured, and easy for a reader to use. | The draft is verbose or ambiguous. |
| Coverage | Addresses the missing context, prerequisites, and maintenance hooks. | Leaves obvious documentation gaps unresolved. |
| Maintainability | Produces docs that can be updated and owned over time. | Creates brittle one-off text. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
