# Release Note Editor

Edit release notes for clarity, audience fit, and upgrade usefulness.

## Quick View

- Category: `docs`
- Maturity: `experimental`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Release notes often become changelog fragments instead of a useful communication artifact.

## When To Reach For It

- Release notes need audience-aware editing.
- The team wants to improve clarity and upgrade guidance.

## What You Should Provide

- draft release notes
- audience
- version context

## What You Should Get Back

- edited release notes
- clarity suggestions
- upgrade highlights

## Repository Notes

- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.
