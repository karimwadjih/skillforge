# Documentation Gap Finder

## Summary

Find the most important gaps in a documentation surface and explain what is missing for real users.

## When To Use

- A documentation area needs a gap audit.
- You need a documentation gap audit with missing topics, user impact, and recommended repair sequence instead of unstructured notes.
- The workflow repeats often enough to justify a reusable asset.

## Avoid When

- The product behavior itself is still changing faster than the docs could stabilize.
- There is no source material to document or clarify yet.

## Inputs

- doc set
- user type
- product or workflow context

## Output Contract

- gap audit
- priority list
- recommended doc additions

## Workflow

1. Confirm the decision context, audience, and missing inputs.
2. Build a documentation gap audit with missing topics, user impact, and recommended repair sequence using the available evidence and constraints.
3. Separate facts from assumptions and make confidence explicit.
4. Finish with next actions, open questions, or escalation notes.

## Deliverable Shape

A documentation gap audit with missing topics, user impact, and recommended repair sequence

## Guardrails

- Do not invent facts that are not present in the supplied materials.
- Make uncertainty explicit when inputs are incomplete or contradictory.
- Keep the output structured enough that another operator can reuse it.

## Maturity Note

This Skill is meaningfully filled for v1, with stronger examples and tests than the broader seeded tier.
