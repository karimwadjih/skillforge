# Documentation Gap Finder

Find the most important gaps in a documentation surface and explain what is missing for real users.

## Quick View

- Category: `docs`
- Maturity: `frontier`
- Tier: `developed`
- Evaluation status: `sampled`

## What This Skill Does

Documentation quality work gets stuck when teams cannot agree on which missing pieces matter most.

## When To Reach For It

- A documentation area needs a gap audit.
- Teams want to know which missing docs matter most to users or contributors.

## What You Should Provide

- doc set
- user type
- product or workflow context

## What You Should Get Back

- gap audit
- priority list
- recommended doc additions

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is meaningfully filled for v1, with stronger examples and tests than the broader seeded tier.
