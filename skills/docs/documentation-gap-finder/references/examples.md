# Examples

## Existing workflow under pressure

### Scenario

A team needs documentation gap finder support for a real deadline instead of a brainstorming session.

### Prompt

Use Documentation Gap Finder to produce a documentation gap audit with missing topics, user impact, and recommended repair sequence for the current workstream.

### Strong Response Should

- State the workflow objective in plain language.
- Produce a documentation gap audit with missing topics, user impact, and recommended repair sequence with explicit structure.
- Call out missing information and the next decision.

## Partial inputs

### Scenario

Inputs arrive incomplete, but the team still needs a useful first-pass artifact from documentation gap finder.

### Prompt

Draft the strongest version of a documentation gap audit with missing topics, user impact, and recommended repair sequence using only the available inputs and list what is missing.

### Strong Response Should

- Separate knowns from assumptions.
- Prioritize the most decision-relevant gaps.
- Keep the output usable even when some inputs are missing.
