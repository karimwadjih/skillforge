# Examples

## Existing workflow under pressure

### Scenario

A team needs service dependency inventory support for a real deadline instead of a brainstorming session.

### Prompt

Use Service Dependency Inventory to produce a dependency inventory with critical dependencies, failure implications, and ownership notes for the current workstream.

### Strong Response Should

- State the workflow objective in plain language.
- Produce a dependency inventory with critical dependencies, failure implications, and ownership notes with explicit structure.
- Call out missing information and the next decision.
