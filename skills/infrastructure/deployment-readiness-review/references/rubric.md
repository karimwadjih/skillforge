# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Gating rigor | Defines which unresolved items block rollout and which can be accepted knowingly. | Lists risks without turning them into go or no-go logic. |
| Rollout realism | Treats flags, staged rollout, and rollback as operational tools with real limits. | Assumes controls exist in theory and therefore solve the problem. |
| Monitoring readiness | Connects launch readiness to the ability to detect, interpret, and respond to failure quickly. | Mentions monitoring without tying it to the actual release risk. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
