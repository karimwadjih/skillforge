# Capacity Planning Brief

Create a capacity planning brief for a service, workflow, or infrastructure area.

## Quick View

- Category: `infrastructure`
- Maturity: `scaffold`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Capacity planning falls apart when assumptions and constraints are not written down in one place.

## When To Reach For It

- Capacity questions need a more organized planning artifact.
- Leaders want a summary of capacity risk and timing.

## What You Should Provide

- usage patterns
- constraints
- growth assumptions
- current limits

## What You Should Get Back

- capacity brief
- risk summary
- planning recommendations

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is part of the seeded breadth layer. It is intentionally lighter than the developed and flagship tiers.
