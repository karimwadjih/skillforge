# Examples

## Existing workflow under pressure

### Scenario

A team needs oncall handoff builder support for a real deadline instead of a brainstorming session.

### Prompt

Use Oncall Handoff Builder to produce an on-call handoff brief with live issues, watch items, timing notes, and escalation reminders for the current workstream.

### Strong Response Should

- State the workflow objective in plain language.
- Produce an on-call handoff brief with live issues, watch items, timing notes, and escalation reminders with explicit structure.
- Call out missing information and the next decision.

## Partial inputs

### Scenario

Inputs arrive incomplete, but the team still needs a useful first-pass artifact from oncall handoff builder.

### Prompt

Draft the strongest version of an on-call handoff brief with live issues, watch items, timing notes, and escalation reminders using only the available inputs and list what is missing.

### Strong Response Should

- Separate knowns from assumptions.
- Prioritize the most decision-relevant gaps.
- Keep the output usable even when some inputs are missing.
