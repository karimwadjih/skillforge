# Infra Change Explainer

Explain an infrastructure change in plain language for operators or adjacent teams.

## Quick View

- Category: `infrastructure`
- Maturity: `experimental`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Infrastructure changes are often well understood by implementers but poorly communicated to everyone else.

## When To Reach For It

- An infrastructure change needs a readable explanation.
- Adjacent teams need to understand an infra change and its impact.

## What You Should Provide

- change description
- affected systems
- audience

## What You Should Get Back

- change explainer
- impact summary
- risk notes

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is part of the seeded breadth layer. It is intentionally lighter than the developed and flagship tiers.
