# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Stage decomposition | Breaks the workflow into reusable stages with explicit handoff contracts and decision points. | Lists steps in order but leaves stage boundaries and responsibilities fuzzy. |
| Failure handling | Names where the workflow can stall, misfire, or require escalation and designs for it. | Acts as if the compiled workflow will only ever run on clean inputs. |
| Package readiness | Produces something that another operator could test, adapt, and ship as a real skill package. | Stops at a process description that still needs major translation work. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
