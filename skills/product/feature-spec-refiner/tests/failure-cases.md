# Failure Case Tests

## Polished ambiguity failure probe

### Prompt

What does a poor Feature Spec Refiner output look like when the document becomes clearer to read but still leaves boundary conditions and ownership unresolved?

### Checks

- Identify the failure mode as polished ambiguity.
- Explain why implementation risk remains high.
- State how the skill should expose unresolved states, dependencies, and decisions.

## Untestable criteria failure probe

### Prompt

What does a poor Feature Spec Refiner output look like when the acceptance criteria sound strong but cannot be verified?

### Checks

- Identify the failure mode as untestable acceptance language.
- Explain how that weakens handoff quality.
- State how the skill should turn goals into concrete conditions.
