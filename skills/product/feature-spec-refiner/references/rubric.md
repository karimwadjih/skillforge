# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Scope sharpness | Distinguishes current scope, deferred scope, and assumptions instead of leaving boundaries implied. | Uses polished language while leaving the actual build boundary fuzzy. |
| Handoff readiness | Surfaces the states, dependencies, ownership questions, and missing decisions that would block implementation. | Reads cleanly but still leaves implementers to discover the hard parts later. |
| Acceptance testability | Turns success criteria into concrete conditions another team could test. | Uses aspirational success language that no one could verify consistently. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
