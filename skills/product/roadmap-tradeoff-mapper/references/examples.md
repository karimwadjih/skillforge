# Examples

## Growth versus reliability quarter

### Scenario

Product leadership has to choose between a self-serve growth initiative and a reliability program that would reduce churn risk.

### Prompt

Compare the self-serve growth initiative with the reliability program, including dependency load, delay cost, and what evidence is strongest today.

### Strong Response Should

- Make the downside of delaying each option explicit.
- Show where the same dependency or team capacity constrains both bets.
- Force a recommendation instead of leaving the tradeoff abstract.

## Three-way platform tradeoff

### Scenario

A PM group is debating admin controls, analytics foundation work, and onboarding improvements for the same release window.

### Prompt

Map the tradeoffs across admin controls, analytics foundation work, and onboarding improvements so the team can decide sequence and cuts.

### Strong Response Should

- Compare the options on the same dimensions instead of changing criteria midstream.
- Surface which option creates future leverage versus immediate user-visible value.
- Recommend sequence or cuts with explicit reasoning.

## Enterprise ask versus core retention

### Scenario

An enterprise feature request could unlock a visible account, but the same quarter also contains a retention bet for the broader base.

### Prompt

Build a tradeoff map between the enterprise request and the core retention initiative, including dependency conflicts, customer concentration risk, and what delaying either choice really costs.

### Strong Response Should

- Show the concentration risk in chasing one account-heavy bet.
- Model dependency conflicts instead of assuming parallel execution.
- Make the recommendation survive executive scrutiny, not just team preference.
