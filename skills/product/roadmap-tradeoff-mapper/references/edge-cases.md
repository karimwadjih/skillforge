# Edge Cases

## Every option leans on the same bottleneck team

### Risk

A platform, data, or security team is a shared dependency, making all bets look independently feasible when they are not.

### How The Skill Should Respond

Model the bottleneck explicitly and show how it changes sequencing, not just desirability.

### Failure Probe

The response ignores the complication or hides the uncertainty instead of adapting the workflow.

## Political favorite has weak evidence

### Risk

One option has executive support but thinner user or business evidence than the alternatives.

### How The Skill Should Respond

Keep the evidence gap visible and state what would justify the favored option more credibly.

### Failure Probe

The response ignores the complication or hides the uncertainty instead of adapting the workflow.

## Cost estimates are uneven

### Risk

One option has detailed sizing while another remains rough, which can distort the comparison.

### How The Skill Should Respond

Note the asymmetry, normalize confidence where possible, and avoid false precision.

### Failure Probe

The response ignores the complication or hides the uncertainty instead of adapting the workflow.
