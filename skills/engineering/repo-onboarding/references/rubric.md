# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Ramp speed | Gets a contributor from clone to a safe first change with minimal dead ends. | Explains the repo at length while leaving the reader unsure where to start. |
| Boundary accuracy | Draws the real runtime boundaries, ownership lines, and dependency edges that matter in practice. | Blurs build-time, runtime, and deploy-time concerns into one vague architecture summary. |
| First-task judgment | Recommends starter work that is useful, low-risk, and aligned with how the repo actually changes. | Ends with generic advice like 'read more code' or proposes work that is unsafe for a newcomer. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
