# Test Plan Designer

Design a test plan for a change, workflow, or system area.

## Quick View

- Category: `engineering`
- Maturity: `experimental`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Teams routinely know what should probably be tested but fail to package it into a reviewable plan.

## When To Reach For It

- A meaningful change needs a structured test plan.
- Teams need to define test scope before implementation or rollout.

## What You Should Provide

- change description
- risk areas
- critical paths

## What You Should Get Back

- test plan
- coverage map
- edge-case list

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is part of the seeded breadth layer. It is intentionally lighter than the developed and flagship tiers.
