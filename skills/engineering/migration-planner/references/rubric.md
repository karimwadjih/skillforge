# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Technical accuracy | Uses repository or system details faithfully and avoids invented architecture. | Guesses at behavior or terminology. |
| Actionability | Produces steps, risks, and follow-ups that engineers can execute. | Stays abstract or descriptive only. |
| Change awareness | Identifies dependencies, rollout risks, and verification steps. | Ignores downstream impact. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
