# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Mental-model accuracy | Gives the audience a correct working model of boundaries, data flow, and dependencies. | Produces a simplified story that breaks the first time someone makes a change. |
| Tradeoff clarity | Names the constraints, bottlenecks, and design tradeoffs that actually shape engineering decisions. | Presents the architecture as neutral description with no decision implications. |
| Audience fit | Matches the level of explanation to the reader and leaves them able to navigate the code or system responsibly. | Either overwhelms newcomers or insults experienced readers with shallow summaries. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
