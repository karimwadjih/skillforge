# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Contract compatibility | Identifies explicit schema breaks and quieter behavioral shifts that consumers still experience as contract changes. | Looks only for obvious type or field removals. |
| Consumer blast radius | Maps which consumers are affected, how badly, and where assumptions are undocumented. | Talks about consumers generically without narrowing the impact. |
| Rollout realism | Pairs the review with deprecation, migration, communication, and support steps that match the risk. | Finds issues but leaves rollout strategy naive or missing. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
