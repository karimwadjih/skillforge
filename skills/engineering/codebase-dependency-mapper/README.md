# Codebase Dependency Mapper

Map important dependencies in a codebase and explain why they matter.

## Quick View

- Category: `engineering`
- Maturity: `scaffold`
- Tier: `seeded`
- Evaluation status: `unbenchmarked`

## What This Skill Does

Dependency risk often stays invisible until changes break more surfaces than expected.

## When To Reach For It

- A team needs a dependency map for a codebase.
- Dependency complexity is making change planning harder.

## What You Should Provide

- repository structure
- dependency manifests
- service boundaries

## What You Should Get Back

- dependency map
- risk notes
- change hotspots

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.

## Quality Position

This Skill is part of the seeded breadth layer. It is intentionally lighter than the developed and flagship tiers.
