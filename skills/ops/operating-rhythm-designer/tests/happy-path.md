# Happy Path Tests

## Founder-led startup scaling happy path

### Prompt

Design an operating rhythm for a scaling startup with weekly execution reviews, monthly planning, and explicit escalation handling across product, engineering, and GTM.

### Checks

- Define which decisions belong in which cadence rather than stacking more meetings.
- Specify the artifact each forum requires to stay useful.
- Make escalation paths explicit when a decision cannot wait for the next scheduled review.

## Distributed cadence cleanup happy path

### Prompt

Create an operating rhythm that reduces duplicated forums, clarifies decision ownership, and matches meeting cadence to data freshness.

### Checks

- Match cadence to how quickly meaningful signal actually arrives.
- Collapse redundant forums instead of renaming them.
- Clarify who decides, who prepares the artifact, and when escalation is triggered.
