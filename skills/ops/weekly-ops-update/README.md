# Weekly Ops Update

Build a weekly operating update that highlights signal, risk, and follow-ups without reading like status spam.

## Quick View

- Category: `ops`
- Maturity: `frontier`
- Tier: `flagship`
- Evaluation status: `benchmarked`

## What This Skill Does

Weekly updates often become activity logs that bury the few issues leaders actually need to track.

## Why This Skill Matters

Ops updates become useless when they read like activity logs. This skill matters when leaders need the operating signal of the week in a format they can scan, trust, and act on.

## What Makes It Different

- Filters for changes in risk, metric movement, and blocked work rather than narrating task completion.
- Links issues to owners and next actions so the update can drive follow-through.
- Keeps the format tight enough for weekly cadence without flattening the hard parts.

## When To Reach For It

- A weekly operating update needs to be synthesized from many small inputs.
- Leaders need the signal from operations without reading every source document.

## What You Should Provide

- weekly operational notes
- metrics
- issues
- cross-functional updates

## What You Should Get Back

- weekly ops update
- key changes
- risks
- follow-up items
