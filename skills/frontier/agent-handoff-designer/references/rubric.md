# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Evaluation rigor | Defines measurable checks rather than vague quality language. | Uses aspirational language without testability. |
| Contract clarity | Makes the workflow, inputs, and outputs explicit enough to reuse safely. | Leaves behavior underspecified. |
| Safety and limits | Explains where the skill can fail and how to handle that safely. | Presents experimental behavior as production-safe by default. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
