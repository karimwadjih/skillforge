# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Behavior coverage | Covers expected behavior, boundary behavior, and failure behavior with distinct cases. | Relies on a handful of generic prompts that miss the risk surface. |
| Scoring observability | Defines dimensions that can be judged from the artifact without hidden grader knowledge. | Uses abstract criteria that different reviewers will interpret wildly differently. |
| Improvement usefulness | Creates eval assets that help maintainers detect regression, explain it, and revise the skill. | Produces more evaluation documents without improving the feedback loop. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
