# Examples

## Internal benchmark kickoff

### Scenario

A team is creating its first benchmark for a new skill library and needs the methodology to be clear before implementation.

### Prompt

Draft a benchmark brief for a new skill-evaluation benchmark, including scope, scoring logic, and failure modes.

### Strong Response Should

- Explain why the benchmark exists.
- Define what the benchmark can and cannot say.
- Link the metric design to decisions the team will actually make.

## Executive result summary

### Scenario

Leadership wants to review benchmark results but should not be overloaded with the full methodology document.

### Prompt

Summarize benchmark results into a short brief that preserves the important caveats.

### Strong Response Should

- Lead with the headline finding.
- Explain the strongest and weakest areas.
- State the most important limitation.

## Version comparison

### Scenario

Two skill versions need to be compared using the same benchmark criteria.

### Prompt

Create a benchmark comparison brief for version A and version B of a workflow skill.

### Strong Response Should

- Compare like-for-like conditions.
- Explain score movement rather than just listing numbers.
- Identify whether the benchmark signals an actual product decision.
