# Positioning Statement Builder

Build a positioning statement with target audience, problem, value, and differentiation.

## Quick View

- Category: `gtm`
- Maturity: `frontier`
- Tier: `developed`
- Evaluation status: `sampled`

## What This Skill Does

Positioning work often stays abstract and never becomes a usable statement with constraints and proof points.

## When To Reach For It

- A product or feature needs clearer positioning.
- Messaging is drifting because the core value statement is not locked.

## What You Should Provide

- audience
- problem
- product value
- alternatives

## What You Should Get Back

- positioning statement
- supporting proof points
- message guardrails

## Repository Notes

- Main instructions live in `SKILL.md`.
- Machine-readable metadata lives in `skill.json`.
- Examples, rubric, edge cases, and tests live under `references/` and `tests/`.
