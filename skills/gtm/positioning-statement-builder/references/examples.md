# Examples

## Existing workflow under pressure

### Scenario

A team needs positioning statement builder support for a real deadline instead of a brainstorming session.

### Prompt

Use Positioning Statement Builder to produce a positioning statement set with audience, value proposition, differentiation, and proof support for the current workstream.

### Strong Response Should

- State the workflow objective in plain language.
- Produce a positioning statement set with audience, value proposition, differentiation, and proof support with explicit structure.
- Call out missing information and the next decision.

## Partial inputs

### Scenario

Inputs arrive incomplete, but the team still needs a useful first-pass artifact from positioning statement builder.

### Prompt

Draft the strongest version of a positioning statement set with audience, value proposition, differentiation, and proof support using only the available inputs and list what is missing.

### Strong Response Should

- Separate knowns from assumptions.
- Prioritize the most decision-relevant gaps.
- Keep the output usable even when some inputs are missing.
