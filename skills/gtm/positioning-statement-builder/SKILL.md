# Positioning Statement Builder

## Summary

Build a positioning statement with target audience, problem, value, and differentiation.

## When To Use

- A product or feature needs clearer positioning.
- You need a positioning statement set with audience, value proposition, differentiation, and proof support instead of unstructured notes.
- The workflow repeats often enough to justify a reusable asset.

## Avoid When

- There is no defined audience, channel, or launch moment yet.
- You need final legal review or brand approval rather than a working draft.

## Inputs

- audience
- problem
- product value
- alternatives

## Output Contract

- positioning statement
- supporting proof points
- message guardrails

## Workflow

1. Confirm the decision context, audience, and missing inputs.
2. Build a positioning statement set with audience, value proposition, differentiation, and proof support using the available evidence and constraints.
3. Separate facts from assumptions and make confidence explicit.
4. Finish with next actions, open questions, or escalation notes.

## Deliverable Shape

A positioning statement set with audience, value proposition, differentiation, and proof support

## Guardrails

- Do not invent facts that are not present in the supplied materials.
- Make uncertainty explicit when inputs are incomplete or contradictory.
- Keep the output structured enough that another operator can reuse it.
