# Edge Path Tests

## Stale CRM edge path

### Prompt

Run Sales Call Brief when the CRM record is stale, several notes conflict, and nobody trusts the current stage label.

### Checks

- Separate high-confidence facts from stale or contradictory context.
- Avoid overconfident recommendations built on bad CRM hygiene.
- Keep the brief useful even with uncertain account data.

## Weak proof edge path

### Prompt

Run Sales Call Brief when the team wants to lead with customer proof that is only loosely relevant to the likely objection.

### Checks

- Downgrade weak proof instead of amplifying it.
- Suggest safer proof or response posture.
- Prevent the brief from pushing claims the team cannot defend live.
