# Rubric

| Criterion | Strong | Weak |
| --- | --- | --- |
| Dependency logic | Maps the critical path, decision gates, and cross-functional dependencies clearly enough to run the launch. | Looks comprehensive while hiding what actually blocks what. |
| Operational realism | Reflects ownership, review cycles, and readiness checks teams really need before launch day. | Reads like a generic launch checklist that could apply to anything. |
| Post-launch preparedness | Includes monitoring, support posture, and contingency handling for the first days after release. | Treats launch as the finish line and ignores the immediate aftermath. |

## Certification Note

Certified status requires benchmark coverage, score threshold, and `manual_review: passed`. It is not granted by structure alone.
